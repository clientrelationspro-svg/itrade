## 客户网站 AI 解析功能开发

### 需求
用户要求在客户管理界面添加"客户网站"字段，并附带"AI解析"按钮，点击后自动分析网站并填充客户信息。

### 实现内容

#### 1. 后端修改
- **模型** (`backend/app/models.py`): 在 `Customer` 模型中添加 `website` 字段 (String(200))
- **API 路由** (`backend/app/routers/ai.py`): 添加 `/api/ai/website/analyze` 端点
  - 接收网站 URL
  - 使用 requests 爬取网页内容
  - 使用 BeautifulSoup 解析 HTML
  - 调用 AI 服务分析内容并提取结构化信息
  - 返回 JSON 格式的客户信息

#### 2. 前端修改
- **客户表单** (`frontend/src/views/CustomerList.vue`):
  - 添加"客户网站"字段 (el-input)
  - 添加"AI 解析"按钮，点击后调用后端 API
  - 添加 `websiteAnalyzing` 状态变量控制加载状态
  - 实现 `analyzeWebsite()` 函数处理 AI 分析逻辑
- **API 接口** (`frontend/src/api/index.js`):
  - 在 `aiAPI` 中添加 `websiteAnalyze()` 方法

#### 3. 数据库迁移
- 创建 `ADD_WEBSITE_FIELD.sql` 脚本，用于手动添加 `website` 字段到 `customers` 表

#### 4. 依赖管理
- 更新 `backend/requirements.txt`，添加 `requests`、`beautifulsoup4`、`lxml` 依赖

#### 5. 文档
- 创建 `DEPLOY_WEBSITE_FEATURE.md` 部署指南

### 技术要点
- 使用 BeautifulSoup 解析 HTML 并提取文本内容
- 限制文本长度为 5000 字符，避免 AI token 过多
- AI 提示词工程：要求返回结构化 JSON 数据
- 处理 AI 响应中的 markdown 代码块格式
