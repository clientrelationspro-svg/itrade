# 2026-06-15 部署日志

## 完成：AI外贸工作平台全栈部署

### 部署结果
- **前端**: https://frontend-ten-teal-98.vercel.app (Vercel)
- **后端**: https://ai-trade-platform-api.onrender.com (Render, Docker)
- **数据库**: Render PostgreSQL

### 关键修复

1. **asyncpg 驱动**: 代码使用 `create_async_engine`，需 `asyncpg`（不是 `psycopg2`）。Dockerfile 添加 `libpq-dev` 系统依赖。

2. **pydantic 依赖冲突**: 
   - `pydantic==1.10.13`（不需要 Rust 编译）
   - 移除 `pydantic-settings`，改用 `pydantic.BaseModel` + `os.getenv()`

3. **数据库初始化**: `lifespan()` 中添加 `await init_db()`，启动时自动建表。

4. **CORS 配置**: 硬编码前端 URL 到 `allow_origins`（`allow_credentials=True` 时不能使用 `["*"]`）。

5. **前端 404 根因（3个）**:
   - `vercel.json` 路由多写一层 `/frontend/dist/` 前缀 → 用 `rewrites` 替代 `routes`
   - `BASE_URL` 缺少 `/api` 后缀 → 硬编码 `...onrender.com/api`
   - Vercel 缓存旧部署 → `npx vercel --prod --yes` 强制刷新

6. **部署方式**: 使用 `npx vercel --prod` 直接部署（绕过 GitHub 集成可能导致的配置冲突）。

### 技术栈
- 后端: FastAPI + SQLAlchemy (async) + asyncpg + Docker (python:3.11-slim)
- 前端: Vue 3 + Vite + Element Plus + Axios
- 部署: Render (Docker) + Vercel CLI
