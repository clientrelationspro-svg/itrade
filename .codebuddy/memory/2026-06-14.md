## 修复 AI 识别 500 错误

### 问题描述
用户报告：新增客户的 AI 识别失败，错误代码是 500 (Request failed with status code 500)

### 根本原因
1. **OCR 响应未正确解析**：`ai_agents.py` 中的 `ocr_fill_form()` 方法直接返回了 AI 原始 API 响应（包含 `choices`、`usage` 等），而没有解析出实际的字段内容
2. **视觉模型不可用**：配置的 `Qwen/Qwen2.5-VL-72B-Instruct` 模型在 SiliconFlow 上被禁用或不存在
3. **缺少错误处理**：没有适当的异常处理，导致任何错误都会返回 500

### 修复内容

#### 1. 修复 `backend/app/ai_agents.py`
- 修改 `ocr_fill_form()` 方法，正确解析 AI 响应内容
- 使用 `_parse_json()` 方法提取字段数据
- 添加异常捕获和日志记录

```python
result = await ai_service.vision_extract_fields(image_data, fields, mime_type)
try:
    content = result["choices"][0]["message"]["content"]
    parsed = AIAgents._parse_json(content)
    return parsed
except Exception as e:
    logger.error(f"Failed to parse OCR result: {e}")
    return {"error": "Failed to parse AI response", "raw": result}
```

#### 2. 修复 `backend/app/routers/ai.py`
- 为 `/ocr/extract` 和 `/ocr/extract-fields` 端点添加 try-except 错误处理
- 捕获异常并返回友好的错误信息

#### 3. 增强 `backend/app/ai_service.py`
- 为 `_chat_completion()` 方法添加详细的错误捕获
- 记录 SiliconFlow API 错误信息（HTTP 状态码、响应内容）

#### 4. 更新 AI 模型配置
- 查询 SiliconFlow API 获取可用模型列表
- 将视觉模型从 `Qwen/Qwen2.5-VL-72B-Instruct`（禁用）更新为 `Qwen/Qwen3-VL-8B-Instruct`（可用）
- 更新 `backend/app/config.py` 和 `backend/.env`

### 测试结果
- 后端成功启动，无错误
- OCR 端点正常响应，返回正确格式的 JSON 数据
- 测试图片成功识别：
  - 公司名称: "ABC Trading Co. Ltd."
  - 邮箱: "zhangsan@abc.com"
  - 电话: "+86-21-12345678"

### 部署状态

## 标注产品新增页面的必填字段

### 需求
用户要求：新增产品是哪个字段不填导致无法保存，请将这个字段标注为必填字段

### 分析结果
根据后端 `ProductCreate` 模型定义：
- **产品名称 (name)**: `str` 类型，**必填**（无 Optional）
- 其他字段均为 `Optional`，可选填

### 实现内容

#### 1. 修改 `frontend/src/views/ProductList.vue`

**添加表单验证规则** (`rules`):
```javascript
const rules = {
  name: [
    { required: true, message: '请输入产品名称', trigger: 'blur' },
    { min: 2, max: 100, message: '产品名称长度应在 2-100 个字符', trigger: 'blur' }
  ],
  // 其他字段的可选验证...
}
```

**修改表单组件**:
- 为 `el-form` 添加 `:rules="rules"` 属性
- 为"产品名称"的 `el-form-item` 添加 `prop="name"` 属性
- 在所有必填字段标签后添加红色星号（Element Plus 自动显示）
- 添加 `placeholder` 提示文本

**添加保存前验证**:
```javascript
async function save() {
  // 先进行表单验证
  try {
    await formRef.value.validate()
  } catch (validationError) {
    ElMessage.warning('请检查表单填写是否正确')
    return
  }
  // ... 保存逻辑
}
```

### 必填字段标识
- ✅ **产品名称**: 红色星号 + "必填" placeholder + 表单验证
- ✅ 其他字段：可选填，但有格式验证（HS编码、MOQ）

### 用户体验改进
1. **视觉提示**: 必填字段显示红色星号
2. **验证提示**: 未填写必填字段时显示错误提示
3. **保存前验证**: 阻止提交不完整的表单
4. **友好提示**: 添加 placeholder 说明字段格式
